main.py — the complete entry point (~20 lines)

# main.py — bootstrap and run. No business logic here. from core.bootstrap import Bootstrap from core.bot_loop import BotLoop if __name__ == "__main__": ctx = Bootstrap.from_env().build() # reads .env + config/bot.yaml BotLoop(ctx).run() # starts loop, delegates to plugins

What "plug and play" looks like in practice

Adding Binance Futures
Create exchanges/binance/__init__.py, implement ExchangeInterface, register as 'binance'. Set exchange: binance in bot.yaml. Strategy, risk, persistence, observability are untouched and work identically.
Adding an RSI-based strategy
Create strategies/rsi_breakout/, implement StrategyInterface, register as 'rsi_breakout'. Set strategy: rsi_breakout in bot.yaml. Both strategies can run as separate bot instances simultaneously with different config files.
Switching WMA-200 → WMA-90 regime filter
In bot.yaml: regime_period: 90. One value. No code change. wma_regime.py reads the period from config at init.
Switching USDT ↔ USDC
In bot.yaml: market: usdc. The Bybit plugin reads market from config to select the correct symbol suffix and quote currency. No second script, no forked code, no deleted file.
Changing partial exit sizes
In bot.yaml: partial_sizes: [0.30, 0.30, 0.40]. One line. Risk module reads from config at runtime.
Migration note — existing DB and tests
The existing SQLite schema is preserved. persistence/db.py runs forward migrations on startup. Existing verify_bot.py and test_live_bot.py move to verification/ with updated import paths. No test logic changes required.
